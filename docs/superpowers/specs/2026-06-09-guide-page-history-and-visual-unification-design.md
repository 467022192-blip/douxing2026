# 攻略页历史入口与视觉统一设计

> 本稿覆盖 `2026-06-09-guide-page-experience-design.md` 中与「攻略页信息架构、历史入口、视觉风格、自动保存交互」相关的决策；旧稿中未冲突的生成等待态与基础性能优化思路仍可继续参考。

## 背景

当前「攻略」页已经具备输入需求、生成 3 套景点攻略、保存历史的基础能力，但仍存在以下问题：

- `攻略` tab 图标辨识度不够，风格与其他 tab 不统一。
- `攻略` 页整体配色偏蓝，与 `我的` 页现有的 Emerald 体系不一致。
- 页面首屏信息偏多，主任务不够聚焦。
- 已保存攻略仍与当前生成页耦合在一起，不利于持续扩展。
- 已登录用户仍需手动点击保存，不符合“生成即沉淀”的产品心智。

本次改动目标是在不推翻现有生成能力和数据结构的前提下，进一步简化「攻略」主页，并把已保存攻略拆成独立的信息层。

## 用户确认的方向

- `攻略` tab icon 改成更像 `👣` 的脚印感样式，但不能直接用 emoji，需要更适配 tab icon 的图形化设计。
- `攻略` tab icon 配色与 `攻略` 页背景、文本颜色统一向 `我的` 页靠拢。
- 本次视觉统一范围只覆盖 `攻略主页 / 我的攻略列表页 / 攻略详情页 / 攻略 tab`，其他主页面仅修明显冲突，不做全站统一。
- `攻略` 页继续简化：
  - 右上角增加 `我的攻略` 入口
  - `我的攻略` 进入独立列表页
  - 列表页点入独立详情页
  - 已登录用户默认自动保存，无需手动点击保存
  - 生成成功后弹窗提示可进入详情页查看
- 攻略生成成功后，主页继续保留本次结果展示，并增加“向下滑动查看”的交互提示。
- 已登录用户每次主动点击生成，都保存一条新的攻略快照，不做覆盖。

## 目标

- 将 `攻略` 主页收敛成“输入需求 -> 生成 -> 成功提示 -> 浏览本次结果”的单任务页面。
- 将已保存攻略拆为独立的列表页与详情页。
- 统一 `攻略` 与 `我的` 的配色语言与文本层级。
- 提升 `攻略` tab 在底部导航中的辨识度和协调性。

## 非目标

- 本次不改动 AI 生成接口的核心结构。
- 本次不新增分享、删除、编辑、再次生成等复杂历史管理能力。
- 本次不改语音输入。
- 本次不重构 Supabase 数据表结构。

## 信息架构

### 路由结构

新增或明确以下路由：

- `/ai-trip-planner`：攻略主页
- `/ai-trip-planner/history`：我的攻略列表页
- `/ai-trip-planner/history/:id`：攻略详情页

### 页面职责

#### 攻略主页

只保留以下能力：

- 输入自然语言需求
- 触发生成
- 展示生成中状态
- 生成成功后提示可查看详情
- 继续展示本次生成结果

不再承担：

- 历史列表浏览
- 已保存记录切换
- 详情阅读主入口

#### 我的攻略列表页

承载：

- 当前用户已保存攻略列表
- 按时间倒序展示
- 点击进入详情页
- 列表页只读取摘要信息，不一次性拉完整结果 JSON

#### 攻略详情页

承载：

- 单条攻略的完整结果浏览
- 原始需求摘要
- 生成时间
- 3 套攻略结果
- 命中景点详情跳转

## 视觉统一

### 总体原则

- `攻略` 页视觉体系向 `我的` 页对齐，而不是继续使用当前蓝色主导方案。
- 页面底色统一优先使用 `bg-gray-50`。
- 内容卡片统一优先使用 `bg-white`。
- 标题文字优先使用 `text-gray-800`。
- 次级说明优先使用 `text-gray-500`。
- 强调色优先复用 `emerald-500` 到 `teal-600` 的渐变体系。

### 底部导航

- `攻略` tab 文案保持 `攻略`。
- `攻略` tab icon 改为脚印感的自定义 icon。
- icon 不使用 emoji 本身，而是使用适配 tab 的抽象脚印造型。
- active 态颜色与 `我的` 页顶部色系一致，优先用 Emerald / Teal。
- inactive 态与其他 tab 一样使用中性灰阶。

### 页面背景和文字统一

本次只改 `攻略相关页面`：

- 攻略主页
- 我的攻略列表页
- 攻略详情页
- 攻略 tab

目标是：

- `攻略` 不再延续当前偏蓝色的独立风格
- 与 `我的` 页共享同一配色语言与文本层级
- 与其他主页面发生明显视觉冲突时，只做局部修正

## 攻略主页改版

### 顶部结构

- 去掉独立的 Hero 卡片模块。
- 标题直接嵌入页面背景，不再单独包成大卡片。
- 页面右上角增加 `我的攻略` 按钮入口。

### 顶部文案

- 主标题：`旅行攻略智能生成`
- 副标题：`说出你的出行天数、景点偏好、出发地等需求`

去掉以下内容：

- `我可以帮你做攻略`
- `结合海量真实景点库生成`
- 当前偏解释性的首屏灵感卡片表达

### 输入区

保留：

- 主输入框
- 1 到 2 个示例问题
- 生成按钮
- 当前本次生成结果区

简化：

- 减少辅助文案占位
- 让输入区成为页面真正视觉中心
- 去掉历史列表与切换入口在正文中的占位

### 结果展示策略

生成成功后主页继续保留本次结果，原因如下：

- 自动保存失败时不会导致用户“无处可看”
- 用户可以立即浏览结果，不必被迫跳详情页
- 详情页主要承担“回看已保存内容”的职责

主页结果区需增加轻提示，让用户知道结果在下方：

- 提示文案建议：`已生成 3 套攻略，向下滑动查看`
- 视觉上让首屏露出第一张结果卡片的上半部分，形成自然滚动暗示

## 自动保存与生成成功提示

### 自动保存规则

#### 已登录用户

- 生成成功后自动触发保存
- 不需要手动点击“保存”
- 自动保存失败不影响结果展示
- 每次用户主动点击生成，都新增一条新的攻略快照
- `重新生成` 也视为一次新的主动生成，新增一条新记录

#### 未登录用户

- 允许正常生成
- 不自动保存
- 不在生成主流程中强制打断去登录

### 成功提示

生成并自动保存成功后，弹出轻量提示层：

- 标题：`攻略已生成`
- 说明：`已为你保存，可前往详情页查看`

操作按钮：

- `查看详情`
- `留在当前页`

### 异常提示

如果生成成功但自动保存失败：

- 仍展示生成结果
- 提示：`攻略已生成，但保存失败，可稍后再试`
- 不强制跳详情页
- 可保留一个轻量的“重新尝试保存”入口

### 生成后状态机

主页至少覆盖以下状态：

1. 生成中
2. 生成成功 + 自动保存成功
3. 生成成功 + 自动保存失败
4. 生成失败

状态约束：

- 生成成功且保存成功：
  - 主页展示结果
  - 弹提示，可去详情页或留在当前页
- 生成成功但保存失败：
  - 主页展示结果
  - 提示保存失败
  - 不展示“去详情页”作为唯一主路径
- 生成失败：
  - 保留用户输入
  - 不生成新历史记录

## 我的攻略列表页

### 结构

- 顶部导航栏
- 列表区
- 空状态

### 列表项字段

每条记录展示：

- 输入摘要
- 创建时间
- 攻略方案数
- 若可行，增加一行简短副标题，如首个方案标题

### 空状态

- 未登录时提示登录后可查看已保存攻略
- 已登录但暂无数据时提示先生成一条攻略

### 数据读取策略

- 列表页默认只拉最近 `20` 条摘要数据
- 如需更多，再通过分页或“加载更多”获取
- 列表接口不应一次性读取完整 `result_json`

## 攻略详情页

### 顶部信息

- 原始需求摘要
- 生成时间

### 内容区

- 复用现有攻略结果卡片结构
- 以阅读体验为主，不再包含输入框
- 保留景点详情跳转

### 数据来源

- 优先根据 `history/:id` 读取已保存攻略详情
- 若数据不存在或用户无权限，展示友好错误态
- 详情页读取完整 `result_json`

### 路由与权限矩阵

需明确以下场景：

- 未登录访问 `/ai-trip-planner/history`
  - 引导登录，不展示用户历史
- 未登录访问 `/ai-trip-planner/history/:id`
  - 引导登录，不展示详情
- 已登录但访问到不属于自己的记录
  - 展示无权限提示
- 已登录但记录不存在
  - 展示记录不存在提示

### 底部导航策略

- `我的攻略列表页` 和 `攻略详情页` 默认不显示底部导航
- 若后续需要保留底部导航，也必须保持 `攻略` tab 高亮
- 本次推荐隐藏底部导航，提升阅读专注度并减少层级干扰

## 数据流

### 生成主链路

1. 用户在攻略主页输入需求
2. 前端请求 AI 生成接口
3. 成功后若用户已登录，则自动调用保存接口
4. 保存成功后返回保存记录 ID
5. 页面弹出提示，允许跳转详情页

失败分支：

- 若自动保存失败，则不生成详情页跳转主链路
- 主页保留结果，并提示稍后再试保存

### 浏览历史链路

1. 用户点击 `我的攻略`
2. 进入列表页读取当前用户历史
3. 点击某条记录进入详情页
4. 详情页按保存内容渲染

## 数据策略

### 记录策略

- 每次用户主动点击生成，都新增一条攻略记录
- 不按 query 内容去重
- 不覆盖旧记录
- 历史记录以“生成快照”形式存在，便于回看不同版本

### 排序策略

- 历史列表按 `created_at desc` 倒序展示

### 字段策略

- 列表页使用摘要字段
- 详情页才读取完整 `result_json`
- 历史列表中的时间展示以 `created_at` 为准

- `GuideTabIcon`：脚印感 tab icon

## 测试与验收

至少验证以下场景：

- `攻略` tab icon 更换成功，active/inactive 态正常。
- `攻略` 页背景与字体颜色与 `我的` 页视觉统一。
- 攻略主页首屏已简化，不再出现旧 Hero 卡片和相关文案。
- 右上角 `我的攻略` 可进入列表页。
- 列表页可进入详情页。
- 已登录用户生成后自动保存成功。
- 生成成功后出现提示层，并可跳转详情页。
- 主页在生成成功后继续保留结果展示。
- 主页有明确的“向下滑动查看”提示或视觉暗示。
- 未登录用户生成不被打断，但不会自动保存。
- 保存失败时不会丢失当前生成结果。
- 已登录用户连续生成多次时，会新增多条历史快照。
- 列表页与详情页的未登录、无权限、记录不存在状态正确。

## 风险

- 自动保存会让“生成成功”和“保存成功”两个异步过程耦合，需要明确异常提示。
- 新增列表页和详情页后，路由与数据加载状态会比当前单页模式更复杂。
- 若页面配色统一处理过大，可能影响其他现有页面的既有视觉习惯。

## 推荐实施顺序

1. 统一 tab icon 与 `攻略` 页配色
2. 简化攻略主页结构与文案
3. 增加 `我的攻略` 列表页
4. 增加攻略详情页
5. 接入自动保存与成功提示层
6. 做登录/未登录及详情跳转验证
