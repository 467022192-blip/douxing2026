# 首页加载性能优化（SWR 缓存 + 轻字段）设计

**Goal**
- 解决首页在弱网下“加载慢/偶发失败”的问题。
- 在不影响搜索与筛选准确性的前提下，提升首屏可用性与稳定性。

## 范围

- 首页（推荐列表）为主；足迹/行程规划/发布页复用同一景区列表接口。
- 搜索词/筛选（想去、去过）仍走实时请求，不使用旧缓存。

## 关键结论

1. 景区列表请求应避免 `select('*')`，改为仅取列表所需字段（减少 payload）。
2. 首页推荐列表引入本地 24 小时 SWR 缓存：先秒出缓存，再后台刷新；失败时继续展示缓存。
3. UI 只在“无数据且请求失败”时展示失败态；若有缓存数据则展示轻提示即可。

## 数据与缓存策略

- 缓存对象：推荐列表（无关键词、无 filterIds、province=全部）。
- 缓存位置：`localStorage`。
- TTL：24 小时。
- 策略：SWR（stale-while-revalidate）。

## 交互

- 冷启动：若有缓存，立即渲染列表；同时显示轻量“正在更新”加载态。
- 弱网失败：如果已有缓存数据，不展示整页失败；保留列表并隐藏强错误。
- 真失败：无缓存且请求失败，展示“加载失败 + 重试”。

## 验收

- 首页推荐列表在弱网下可更快出现内容（缓存命中时无需等待接口）。
- 搜索/筛选不使用缓存，仍实时请求。
- `lint/check/build` 通过，线上不出现新增控制台错误。
