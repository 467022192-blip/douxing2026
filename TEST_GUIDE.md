# 空间页面测试指南

本文档说明如何测试空间页面的点赞和评论功能。

## 🧪 测试方法

### 方法一：浏览器控制台手动测试（推荐）

1. 打开浏览器开发者工具（F12）
2. 切换到 Console 面板
3. 运行以下测试命令：

#### 测试场景1: 点赞功能测试
```javascript
testScenarios.likeFlow()
```

**预期结果：**
- 控制台显示"用户已登录: 点赞测试用户"
- 按照提示手动测试点赞功能
- 验证点赞数是否正确变化
- 验证按钮颜色是否正确变化

#### 测试场景2: 评论功能测试
```javascript
testScenarios.commentFlow()
```

**预期结果：**
- 控制台显示"用户已登录: 评论测试用户"
- 按照提示手动测试评论功能
- 验证评论弹窗是否正常打开
- 验证评论是否能正常提交

#### 测试场景3: 完整用户场景测试
```javascript
testScenarios.fullScenario()
```

**预期结果：**
- 模拟完整的用户使用流程
- 包括登录、浏览、点赞、评论等操作

#### 测试场景4: 未登录用户测试
```javascript
testScenarios.guestScenario()
```

**预期结果：**
- 验证未登录状态下功能是否正常

#### 其他辅助命令
```javascript
// 手动登录
testScenarios.login({ nickname: '自定义用户名' })

// 手动登出
testScenarios.logout()

// 获取当前认证状态
testScenarios.getAuthStatus()
```

### 方法二：单元测试

运行单元测试验证功能：

```bash
# 安装测试依赖
npm install -D vitest @testing-library/react @testing-library/jest-dom

# 运行测试
npm run test
```

测试文件位置：
- `src/stores/__tests__/auth.test.ts` - 认证状态测试
- `src/pages/Space/__tests__/Space.test.tsx` - 空间页面功能测试

## 🔍 常见问题排查

### 问题1: 控制台提示 `testScenarios is not defined`
**解决方案：**
- 确保在开发环境下运行（`npm run dev`）
- 刷新页面后重试

### 问题2: 点赞/评论状态不保持
**解决方案：**
- 检查浏览器 localStorage 是否正常
- 清除缓存后重试

### 问题3: 评论弹窗不显示
**解决方案：**
- 检查是否有 JavaScript 错误
- 查看控制台错误信息
