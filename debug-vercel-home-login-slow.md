# [OPEN] vercel-home-login-slow

## 症状

- 线上 `https://douxing2026.vercel.app/` 首页加载慢，甚至失败；登录也慢/失败。
- 本地预览加载较快。

## 假设（可证伪）

1. **网络层/区域访问问题**：线上到 Supabase 或图片 CDN 的请求在某些网络下超时/被拦截，导致首页等待或错误。
2. **数据请求过重或频繁**：线上触发多次景区列表请求或请求体过大，造成慢与失败。
3. **Supabase Auth 端点不稳定**：登录请求在 Vercel 线上环境更容易超时/失败（例如跨境/运营商问题）。
4. **前端错误导致重试/中断**：线上出现 JS runtime error 或请求被频繁 abort，导致“看起来像失败”。
5. **缓存策略缺失/失效**：Vercel 线上未命中本地缓存或缓存写入失败，导致每次都冷启动请求。

## 证据采集计划

- 使用浏览器自动化采集线上首页 Network/HAR、关键请求耗时与错误码。
- 若证据不足，再在代码中增加最小化埋点（上报到本地 Debug Server）。

## 修复策略候选

- 轻字段查询 + 推荐列表 SWR 缓存（已实现）。
- 对 Supabase 域名 preconnect、对失败请求添加重试与降级。
- 替换/代理不稳定图片源，或增加图片失败兜底。
